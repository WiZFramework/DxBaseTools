セットアップ

ツール作成の準備


コンパイラと入力インターフェイス

　DxBaseToolsはVisuakStdio2013でツール作成を行えます。Windows8.1、Windows10で動作します。
　もしお持ちでない方はMicrsoft社のサイトから、Visual Studio Community 2013 をダウンロードしてください。
　また、セットアップの際VisualC++とMFCとWindowsSDKを選択してください。

ダウンロードと解凍

　DxBaseToolsはGitHubに公開されています。
　アドレスは https://github.com/WiZFramework/DxBaseTools
　です。
　上記サイトにアクセスすると、DownloadZIPというボタンがありますので、そこで最新版をダウンロードしてください。解凍すると以下のようなディレクトリ構成になります。 
DxBaseTools
　　　　Documents
　　　　Fbx2Bin
　　　　Libs
　Documentディレクトリには、このページを含めたドキュメントが入ってます。index.htmlをブラウザで開いてください。
　Libsディレクトリには、ライブラリや、DirectXTKライブラリなどが入ってます。
　Libs/includeとLibs/libディレクトリには、このあと、Autodesk社のFBX SDK（FBX Software Development Kit）のヘッダとライブラリとコピーしていただきます。（FBXSDKのセットアップで解説します）
　Fbx2Binなどのディレクトリには、ツールが入ってます。現時点でツールはFbx2Binのみです。

FBXSDKのセットアップ

　GitHubからのダウンロードには、FBXモデルの読み込みのためのライブラリが含まれてません。以下の手順でセットアップしたのち、必要なファイルを、DxBaseTools内に配置して利用します。
  • Autodesk社のFBX SDK（FBX Software Development Kit）をセットアップします。
  • FBX SDKは検索エンジンなどで　FBX SDK　などで検索するとたどり着けます。そこからダウンロードのページに移り、VisualStdioS2013に合わせたバージョンのFBX_SDKをダウンロードしてください。（ご使用のコンパイラのバージョンと、FBX_SDKのバージョンが合わないと正常にビルドできませんので注意してください）。
  • 現在、動作が確認されているFBX_SDKのバージョンは、FBX SDK 2015.1 VS2013、FBX SDK 2016.0 VS2013 、FBX SDK 2016.1 VS2013 です。
  • 2015年8月時点でのFBX SDKの最新バージョンはFBX SDK 2016.1です。
  • FBX SDKは実行ファイルになっていますので、そのままセットアップ可能です。
  • FBX SDKのセットアップが済んだら、セットアップされたディレクトリ
    （デフォルトで「C:\Program Files\Autodesk\FBX\FBX SDK\2016.1」などにエクスプローラで移動してください。
  • FBX SDKがセットアップされたディレクトリの「include」ディレクトリ内の、fbxsdk.hというファイルと、fbxsdkディレクトリを、そのまま、DxBaseToolsを解答したディレクトリの「Libs\include」内にコピーしてください。
  • FBX SDKがセットアップされたディレクトリの「lib\vs2013\x86\debug」あるいは「lib\vs2015\x86\debug」ディレクトリ内の、libfbxsdk-mt.libというファイルと、libfbxsdk.pdbを、DxBaseToolsを解答したディレクトリの「Libs\lib\Debug」内にコピーしてください。
  • FBX SDKがセットアップされたディレクトリの「lib\vs2013\x86\release」あるいは「lib\vs2015\x86\release」ディレクトリ内の、libfbxsdk-mt.libというファイルを、DxBaseToolsを解答したディレクトリの「Libs\lib\Release」内にコピーしてください。
  • x64（64ビット版）には対応しておりません。必ずx86（32ビット版）をセットアップしてください。

　以上で、セットアップは完了です。

ツールのの起動

　各ツールディレクトリにはソリューションファイル(.sln)が入っています。Fbx2BinであればFbx2Bin.slnです。
　ソリューションファイル(.sln)は、VisualStdio2013用のソリューションになっています。
　また、MFCが使える環境が必須ですので、注意してください。
